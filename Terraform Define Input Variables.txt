How to define input variables
You create them using a variable block:

variable "instance_type" {
  type        = string
  description = "EC2 instance type"
  default     = "t2.micro"
}



Create vairbales file and use it:


Files in the setup
1. main.tf
hcl
resource "local_file" "pet" {
  filename = var.filename
  content  = var.content
}

resource "random_pet" "my-pet" {
  prefix    = var.prefix
  separator = var.separator
  length    = var.length
}
This file defines two resources:

A local file that writes text to /root/pets.txt

A random pet generator that creates a random name like Mrs.tiger

Each resource references variables using var.<name>.

2. variables.tf
hcl
variable "filename" {
  default = "/root/pets.txt"
}

variable "content" {
  default = "We love pets!"
}

variable "prefix" {
  default = "Mrs"
}

variable "separator" {
  default = "."
}

variable "length" {
  default = "1"
}
This file defines default values for all variables used in main.tf.

How Terraform uses these files
Terraform reads both files — it merges them into one configuration.

When you run terraform apply, Terraform:

Creates /root/pets.txt with the content "We love pets!"

Generates a random pet name like Mrs.tiger

The random name is stored in the state file and can be printed with an output block.

🧠 Why this structure matters
variables.tf makes your configuration reusable — you can override defaults in a .tfvars file or CLI.

main.tf stays clean — it only references var.<name>.

You can easily switch environments by changing variable values.

✅ Typical workflow
Step	Command	Purpose
1	terraform init	Load providers
2	terraform plan	Preview what will be created
3	terraform apply	Execute and create resources
4	terraform state show random_pet.my-pet	View generated name

Example of the variable This defines a single variable named bella that contains multiple attributes — like a mini data structure.

variable "bella" {
  type = object({
    name         = string
    color        = string
    age          = number
    food         = list(string)
    favorite_pet = bool
  })

  default = {
    name         = "bella"
    color        = "brown"
    age          = 7
    food         = ["fish", "chicken", "turkey"]
    favorite_pet = true
  }
}

How to use it in the resource:

resource "local_file" "pet_info" {
  filename = "/root/pet_info.txt"
  content  = "Name: ${var.bella.name}, Color: ${var.bella.color}, Age: ${var.bella.age}"
}


You can override the default values in a .tfvars file:

bella = {
  name         = "luna"
  color        = "white"
  age          = 3
  food         = ["salmon", "beef"]
  favorite_pet = false
}

Then run:terraform apply -var-file="ded.tfvars"
Terraform will use these new values instead of the defaults.

Variable Definition Precedence in Terraform
Terraform allows variables to be defined in multiple places, but when the same variable is set in more than one location, Terraform follows a strict order of precedence to decide which value to use.

Code setup
main.tf
hcl
resource "local_file" "pet" {
  filename = var.filename
}
variables.tf
hcl
variable "filename" {
  type = string
}
These files define a resource that writes to a file, and a variable named filename whose value will be provided externally.



Ways to define variable values
Terraform can get the value for filename from several sources:

Source	Example	Description
Command line flag	terraform apply -var "filename=/root/best-pet.txt"	Highest precedence — overrides all other sources.
Environment variable	export TF_VAR_filename="/root/cats.txt"	Second highest — used if no CLI flag is provided.
terraform.tfvars	filename = "/root/pets.txt"	Automatically loaded if present.
variable.auto.tfvars	filename = "/root/mypet.txt"	Automatically loaded, same level as .tfvars.
Default value in variable block	default = "/root/default.txt"	Lowest precedence — used only if nothing else is provided.
