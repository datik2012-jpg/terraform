Example of local-exec:


resource "aws_instance" "webserver" {
  ami           = "ami-0edab43b6fa892279"
  instance_type = "t2.micro"

  provisioner "local-exec" {
    command = "echo Instance ${aws_instance.webserver.public_ip} Created! > /tmp/instance_state.txt"
  }
}

aws_instance "webserver" — creates an EC2 instance using the specified AMI and instance type.

provisioner "local-exec" — runs a command on your local machine (not inside the EC2 instance) after Terraform finishes creating the resource.

The command writes the instance’s public IP address into /tmp/instance_state.txt, confirming creation.

⚙️ How it works
Terraform executes the local-exec provisioner after the EC2 instance is successfully created.

It interpolates ${aws_instance.webserver.public_ip} — Terraform fetches the instance’s public IP from AWS and substitutes it into the command.

The result is a local file containing something like:

Code
Instance 3.97.9.249 Created!

Examle on if fail to run exec and get ip - will mark terraform resource as failed and during destroy:


What Each Provisioner Does
First local-exec (creation)

Runs after the EC2 instance is successfully created.

Writes the instance’s public IP and a “Created!” message to /temp/instance_state.txt.

The on_failure = fail ensures Terraform stops if this command fails.

Second local-exec (destruction)

Runs when Terraform destroys the instance (terraform destroy).

Logs the instance’s public IP and “Destroyed!” message to /tmp/instance_state.txt.


resource "aws_instance" "webserver" {
  ami           = "ami-0edab43b6fa892279"
  instance_type = "t2.micro"

  provisioner "local-exec" {
    on_failure = fail
    command = "echo Instance ${aws_instance.webserver.public_ip} Created! > /temp/instance_state.txt"
  }

  provisioner "local-exec" {
    when    = destroy
    command = "echo Instance ${aws_instance.webserver.public_ip} Destroyed! > /tmp/instance_state.txt"
  }
}

❌ Terraform stops the apply
❌ Terraform marks the resource as “tainted”
❌ Terraform does NOT destroy the EC2 instance automatically
✔️ The EC2 instance still exists in AWS

The AMI ID we have used is an RHEL 8 image in the London region that only accepts SSH-Key based authentication. However, when we created the instance, we did not make use of a key!

Let's create a new key-pair!



A new SSH key pair has been created in the directory /root/terraform-projects/project-cerberus/.ssh.

The private key is called cerberus and the public key is called cerberus.pub


Using the public key, create a new key-pair in AWS with the following specifications:

Resource Name: cerberus-key

key_name: cerberus

Use the file functions to read the public key cerberus.pub


When ready, run a terrafom plan and apply to create this key pair.


If unsure, refer to the documentation to create a key-pair. Documentation tab is available at the top right.


Private key: /root/terraform-projects/project-cerberus/.ssh/cerberus

Public key: /root/terraform-projects/project-cerberus/.ssh/cerberus.pub


Create an AWS key pair using your public key

resource "aws_key_pair" "cerberus-key" {
  key_name   = "cerberus"
  public_key = file("/root/terraform-projects/project-cerberus/.ssh/cerberus.pub")
}

What this does
Terraform reads your public key file.

Terraform uploads it to AWS.

AWS stores it as a key pair named cerberus.

Any EC2 instance using key_name = "cerberus" will inject your public key into ~/.ssh/authorized_keys.


Attach the key pair to your EC2 instance
Update your EC2 resource:

resource "aws_instance" "cerberus" {
  ami = "ami-06178cf087598769c"
  instance_type = "m5.large"
  key_name      = aws_key_pair.cerberus-key.key_name
  tags = {
    Name        = "webserver"
    Description = "An Nginx WebServer on Ubuntu"
  }
}

We use the public IPv4 address to access this server. However, when this server is rebooted or recreated, this IP address would change.

To fix this, let's create an Elastic IP Address.

An Elastic IP address is a static IPv4 address which does not change over time.


Create an Elastic IP resource with the following specifications:

Resource Name: eip
vpc: true
instance: id of the EC2 instance created for resource cerberus (use a reference expression)
create a local-exec provisioner for the eip resource and use it to print the attribute called public_dns to a file /root/cerberus_public_dns.txt on the iac-server.

If unsure, refer to the documentation. Documentation tab is available at the top right.

Added aws_eip:


resource "aws_instance" "cerberus" {

resource "aws_eip" "eip" {
  vpc      = true
  instance = aws_instance.cerberus.id

  provisioner "local-exec" {
    command = "echo ${aws_eip.eip.public_dns} > /root/cerberus_public_dns.txt"
  }
}
