/////////////////////// Resource Dependencies ////////////


What an explicit dependency is
Terraform normally figures out dependencies automatically by analyzing references between resources.
But sometimes you need to force Terraform to wait for something even if there is no direct reference.

That’s where depends_on comes in.

You add a depends_on block inside a resource, module, or output.

Example:

resource "aws_iam_role" "example" {
  name = "my-role"
}

resource "aws_lambda_function" "example" {
  function_name = "my-func"
  handler       = "index.handler"
  runtime       = "nodejs18.x"

  depends_on = [
    aws_iam_role.example
  ]
}


You use depends_on when:

Implicit dependencies don’t exist

A resource depends on side effects (e.g., IAM policies, permissions)

A module must wait for another module

You need to enforce a creation order for safety or correctness

You want to avoid race conditions in cloud providers

Explicit dependency on modules
You can also apply it to modules:

module "network" {
  source = "./network"
}

module "servers" {
  source = "./servers"

  depends_on = [
    module.network
  ]
}
