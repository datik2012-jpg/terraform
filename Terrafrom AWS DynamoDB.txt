DynamoDB Attribute Types
1. S — String
Used for text values like names, emails, IDs.

2. N — Number
Used for integers, floats, timestamps.

3. B — Binary
Used for raw bytes (rare unless storing encoded data).

DynamoDB Set Types (collections)
4. SS — String Set
Example: ["red", "blue", "green"]

5. NS — Number Set
Example: [1, 2, 3]

6. BS — Binary Set
Example: [<binary1>, <binary2>]

Terraform Example Using All Types



DynamoDB Table with id, name, secondname
hcl
resource "aws_dynamodb_table" "pixar_people" {
  name         = "pixar-people"
  billing_mode = "PAY_PER_REQUEST"

  # Primary key (required)
  hash_key = "id"

  # Attributes
  attribute {
    name = "id"
    type = "S"
  }

  attribute {
    name = "name"
    type = "S"
  }

  attribute {
    name = "secondname"
    type = "S"
  }

  tags = {
    Environment = "dev"
    Owner       = "dani"
  }
}

resource "aws_dynamodb_table" "project_sapphire_inventory" {

    hash_key         = "Hardware"
    projection_type    = "ALL"
    
  }
}
